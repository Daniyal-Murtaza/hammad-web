「上帝粒子」物理學家、資料工程學者 Gogoro 用數學模型蓋電池交換站

Gogoro 想太多

曾在歐洲核子研究組織(CERN)研究「上帝粒子」的物理博士、來自世界最大地理資訊系統公司 ESRI 的資料工程師，還有海外名校畢業的統計學家, 地理資訊顧問和數據分析專家等高手；Gogoro Network 在哪裏蓋 GoStation 電池交換站、蓋多少的理論依據，背後就是由這個學經歷驚人的團隊，以數學模型和大數據演算出來，更開發出人工智能選點系統，自動分析電池交換站最適點位。

Gogoro Network 是全球最大智慧電動機車電池交換服務平台，目前在台灣擁有超過 40 萬用戶、2186+ 站電池交換站以及流通使用超過 80 萬顆智慧電池，其中決定 PBGN (Powered by Gogoro Network) 電動機車續航關鍵的電池交換站位址，並非憑空選出，都是內部學者團隊的研究成果，我們首度曝光這個神秘部門、以及 Gogoro Network 電池交換站的佈建邏輯。

六年大數據、數十套數學模型 開發智能選點系統

Gogoro Network 研究團隊獨力開發出選點系統，命名為「司馬駿(SmartGEN)全智慧五層能源網路佈建演算法」，此系統原理，是透過地理資訊蒐集與判讀, 搭配用戶問卷、用戶使用行為分析等，演算出用戶能輕鬆找到電池交換站的最適距離，能輕鬆規劃區域需要的合理電池交換站數、以及掌握站點的位置和效率(如何選點能讓用戶最容易拿到滿格電池)。

 為開發演算法，Gogoro Network 的大數據分析團隊，運用及建立過協同過濾(COLLABORATIVE FILTERING)、K-平均演算法(K-MEANS CLUSTERING)、蒙地卡羅法(MONTE CARLO)等幾十套以上的數學模型，而其中需要的參數來自於各種 Gogoro Network 蒐集的去識別化大數據，包括充電時間、電池種類、電池使用和換電池次數、電池交換站供給的電力、人口、車輛數、地理資訊、車種、耗電效率…等數十種參數整合而成。

在參數中，最重要的就是用戶使用數據，像騎乘行為、交換電池的地點次數等，這些都是 Gogoro Network 智慧系統獨有的記錄，已經累積六年數據，所以團隊可以據此不斷改進運算模型，持續優化。

「我們每週都會重新檢視各站點效率及尋找潛在站點，讓演算法和電池交換站都能根據最新現況有最佳使用。」領導 Gogoro Network 分析團隊的資深經理說。

 

分析單位超細 智能系統連國外也可評估

Gogoro Network 選點系統的演算力驚人，像是在評估站點時，數學模型內最小的分析單位為每 500 人、比台灣最小行政區還細緻；舉例來說，一個彰化縣和美鎮裡，可以運算到每 500 人的交換量、人口密度、地理環境、車輛領牌數、車流等等，去評估潛在或現況需要的電池交換站範圍。

簡言之，以這個等級運算 Gogoro Network 大規模電池交換網路下的使用者行為數據，經大量研究及跑迴歸分析，能發現什麼因素會影響電池交換量高低，建構出評分機制，透過電腦建置成人工智慧系統，就成為「司馬駿全智慧五層能源網路佈建演算法」。

自動化後，在電腦上輸入任一或數個地點，Gogoro Network 可以快速評估當地電池交換站建置，不只台灣，只要改動人口及地理位置等參數，就連國外都能直接顯示該區域是否適合蓋站、或需要多少站，且系統精確度還會隨數據更新不斷成長。

全台電池交換站覆蓋 8 成 5 騎士換到滿格電池的最佳解：Super GoStation 超級電池交換站 

透過「司馬駿全智慧五層能源網路佈建演算法」，Gogoro Network 六年多來建了超過2千個站點，讓全台灣 85% 的機車騎士平均能在騎乘 5 分鐘之內換到電池，而六都則是超過九成的機車騎士可辦到 (台北市高達 99%)。也就是說，只要你使用機車，不管是燃油或電力驅動，要找到 Gogoro Network 電池交換站都很方便。

通常在尖峰時刻的精華路段，換電池需求最大，而現實上，精華路段因為交通法規、坪效以及店家營業空間的珍稀，也最難建電池交換站。所以演算法找出的最佳方法，就是在能覆蓋精華路段換電範圍的區域，加蓋提供更多電池的 Super GoStation ，並且優化 App，推薦用戶哪些範圍內的其他站點還有更多滿格電池。

據統計，Gogoro Network 從 2020 年開始蓋 Super GoStation 、加上檢討交換電池缺口，滿格電池的供給量持續提高，2021年底 Super GoStation 會從現有的 100 站提升到 140 站，且電池交換站都是特別從台電申請較大的電力，在充電效率上更高，預期用戶會更容易能拿到滿格電池。

比起沒有策略的隨意建站， Gogoro Network 透過精密計算找解答，更能在需要的位置滿足交換電池需求，且經這六年來的用戶反饋，發現演算法和現實需求的偏差小，還能因應數據的增加不斷演進、隨時調整。
